# Development Runbook

This guide covers running the core platform services locally on your machine for feature development.

## Prerequisites

- Node.js (v18+)
- NPM
- A verified checkout of the monorepo

## Starting the Platform

The entire ecosystem is managed by Nx. There are two primary services you need to spin up to view the application:

1. **The Backend REST Proxy (NestJS)**
   Open a terminal and run:
   ```bash
   npx nx serve api
   ```
   *This starts the server on `http://localhost:3000`.*

2. **The Frontend Single Page App (Angular)**
   Open a second terminal and run:
   ```bash
   npx nx serve portal
   ```
   *This starts the Angular dev server on `http://localhost:4200`.*

Navigate to `http://localhost:4200` in your browser. The Angular dev server automatically proxies all `/api/*` traffic to your local NestJS backend.

## Understanding Render Modes

### Local Mode (Dynamic)
When running the `npx nx serve portal` command natively, the frontend attempts to read Markdown files dynamically. We utilize `DOMPurify` to simulate exactly what the end-state DOM would look like.

### Production Mode (Prebuilt JSON)
When the `SERVE_PREBUILT_HTML="true"` environment variable is set in the `.env` file, the backend flips its logic. Instead of parsing Markdown on the fly, it serves the highly optimized `.html.json` pre-compiled payloads generated by the CLI.
